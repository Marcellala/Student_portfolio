Overview

Problem

A small manufacturing organization has outgrown its current paper-based inventory system and would like a more sophisticated program developed. The inventory is categorized into ‘parts’ and ‘products’ where products are made from parts. The requirements include adding, modifying, and deleting items from inventory.

Solution

The application was built in Java/JavaFX using IntelliJ as an IDE. The GUI component was constructed in SceneBuilder as per GUI specifications. There is no database connection so parts and products are treated as Java objects.

Main Screen

The GUI at a glance

This screen provides the user with a quick snapshot of inventory and an interface to execute the bulk of their work.

From here, the user can add, modify, search for, or delete parts and products. The user can also exit the program at anytime.

Adding Items

Instantly add all items to the application

Each part is either in-house or outsourced and added to the system along with key information.

When a product is added, associated parts can be linked to the product. For example, a bike is a product that might have brakes and wheels associated with it.

Warnings

Dialogues and warnings prevent user error

The application ensures that a product is not deleted unless all associated parts are removed first.

The program prevents adding incomplete items and confirms prior to deleting an item.
