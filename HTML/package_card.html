Overview

Problem

WGUPS needs to determine an efficient delivery distribution algorithm for their daily deliveries and meet promised deadlines. The company has an average of 40 packages a day that need to be delivered by 2 drivers with 3 trucks available for use. Each package has specific criteria and delivery requirements that must be met. The total miles traveled needs to be recorded and come in under 140 miles. The user must be able to see the status of the package at any given time. This assignment was based on the 'Traveling Salesman Problem'.

Solution

The solution built is a CLI application that meets all requirements. The package information is stored in a hash table for easy access. This is a chaining hash table which allows for faster lookups. With the hash table, the package information can be deleted, looked up, and additional packages can be inserted. Each package can be accessed with a unique key, the package ID.

The solution implements a self-adjusting ‘Nearest Neighbor’ algorithm that decides what order to deliver a list of packages based on next closest proximity. The program delivers all 40 packages in under 140 miles while accounting for wrong addresses, delivery deadlines and delays. Additionally, the user interface allows the user to check the status of all packages at any given time or see the status of a specific package.

Time and Space complexities are used to evaluate the algorithms efficiency. The time complexity for the entire program is O(N^2) and the total space complexity for the entire program is O(n).

CLI Application

Package status available at any time

The application begins displaying the total miles traveled, 99.6. Then, the user has the option to enter a time to view the status of all packages at that time or to view a specific package status by entering a package ID.

The program returns either a list of all packages and their status at the input time or returns the requested package by ID and displays package information.
