Overview

Problem

This capstone projects mimics a real world business problem and solution. The fictional company Simmer, is more or less Google for recipes. It does not offer anything more than the ability to search for a recipe by name and provide that recipe or, recipes with similar titles. The proposed solution is to create a content-based recommendation system that will provide the user with the top-rated recipes based on their dietary restrictions. This will serve as the first step for Simmer to begin using machine learning to offer more personalized content to their users and help increase user retention.

Solution

The program was developed in Python using Jupyter notebook and several Python libraries. The benefit to using the Jupyter Notebook is the interactive capabilities for data science, machine learning, and code. By implementing the program in Jupyter, the user can simply enter their dietary restrictions and receive the recommendations all in one convenient location.

Analyzing the Data

Transforming data into useful information to make informed decisions

The recommendation system utilized descriptive methods to analyze the dataset. This information was used to glean insights for the necessary functionality of the predictive outputs. It was also used to determine which machine learning algorithms would best suit the project’s requirements.

To begin, a simple count plot was generated to compare the number of recipes in each ‘diet’ category. Next, a pie chart and histogram were generated to view and compare the different recipe categories. Lastly, a scatterplot was generated to visualize the relationship between the Number of Votes and Ratings.

Machine Learning

Instantly add all items to the application

In addition to the descriptive methods, the program also utilized two non-descriptive methods to implement the machine learning aspect of the problem. The first method used was the decision tree to make predictions. This method was chosen to increase user retention by predicting the type of diet the user has.

Once the dataframe was properly configured for the decision tree, a pair plot was created to visualize the numerical correlation of the Plant Based (diet) column and the other columns across an x and y-axis. This was particularly useful in determining which columns to train the model with.

Finally, the decision tree was implemented and trained to predict the diet of the user. The classification showed an accuracy rate of 68%.

Insightful Recommendations

Dialogues and warnings prevent user error

Lastly, the recommendation system and GUI were built after the decision tree verified the model was trained and provided the necessary accuracy, precision, and F1 scores for dietary prescription.

The recommendation system was based on the relationship between ratings and vote count. Rather than simply suggesting recipes based on highest rating, it was important to take the number of votes into account. This way, a 5-star recipe with only a few votes would not be suggested.

Instead, the program implements an equation based on the Bayesian rating equation. This equation is particularly beneficial when working with large datasets. Once the computation is complete, the top 5 recipe recommendations are sorted and printed for the user to view.
